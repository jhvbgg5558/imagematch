# RoMa v2 在 Intersection Truth 口径下的重排结果说明

## 1. 任务定义与实验设置
本组实验用于回答：在 `intersection truth` 新真值口径下，若先用 `DINOv3 + FAISS` 对全库粗检索，再对前 `Top-20` 候选使用 `RoMa v2` 做精匹配重排，是否能够进一步改善正式指标。

- 数据范围：4 条航线，共 `40` 个 query。
- 粗检索：DINOv3 全局特征 + FAISS `IndexFlatIP`。
- 重排方法：RoMa v2 dense matching + 采样点对 + RANSAC 几何一致性 + 融合分数排序。
- 重排窗口：每个 query 的 coarse `Top-20`。

## 2. 总体定量结果

- Baseline：`R@1=0.775`，`R@5=0.950`，`R@10=1.000`，`R@20=1.000`，`MRR=0.850`
- Coarse Top20 上限：`R@20=1.000`
- RoMa v2：`R@1=0.925`，`R@5=1.000`，`R@10=1.000`，`R@20=1.000`，`MRR=0.958`
- 指标变化：`ΔR@1=+0.150`，`ΔR@5=+0.050`，`ΔR@10=+0.000`，`ΔR@20=+0.000`，`ΔMRR=+0.108`
- Top-1 误差：`862.191m -> 630.313m`，`Δ=-231.878m`

## 3. 分航线结果

- `009`：Baseline `R@1=0.700`、`MRR=0.798`；RoMa v2 `R@1=0.900`、`MRR=0.933`；`ΔR@1=+0.200`、`ΔMRR=+0.136`。
- `010`：Baseline `R@1=0.900`、`MRR=0.950`；RoMa v2 `R@1=1.000`、`MRR=1.000`；`ΔR@1=+0.100`、`ΔMRR=+0.050`。
- `011`：Baseline `R@1=0.700`、`MRR=0.803`；RoMa v2 `R@1=0.900`、`MRR=0.950`；`ΔR@1=+0.200`、`ΔMRR=+0.147`。
- `012`：Baseline `R@1=0.800`、`MRR=0.850`；RoMa v2 `R@1=0.900`、`MRR=0.950`；`ΔR@1=+0.100`、`ΔMRR=+0.100`。

## 4. 时间开销统计

- RoMa v2 重排：`9049.65s (150.83 min)`

## 5. 代表性样例

- `q_001`：前排排名进一步提升样例；Baseline 首个真值 rank=`7`，Coarse Top20 rank=`7`，RoMa v2 rank=`3`。

## 6. 结论

- 本轮 `RoMa v2` 相对 `DINOv3 baseline` 的主要收益集中在前排排序：`ΔR@1=+0.150`、`ΔR@5=+0.050`、`ΔMRR=+0.108`，说明精匹配显著提升了前几名候选的判别力。
- `Top-1 error mean` 从 `862.191m` 降至 `630.313m`，改善 `231.878m`，说明这轮重排不仅改善了排名，还明显提升了首位候选的空间精度。
- 四条航线的 `R@1` 均未低于 baseline，其中 `010` 航线达到 `1.000`，其余三条航线均达到 `0.900`，说明收益具有跨航线稳定性。
- 在当前 `query v2 + intersection truth` 口径下，`RoMa v2` 可以视为一条有效的几何重排方法，显著优于 `DINOv3` 粗检索 baseline。
