# DINOv3 + FAISS 在 Intersection Truth 口径下的基线实验结果说明

## 1. 任务定义与实验设置
本组实验用于回答：在新的 `intersection truth` 真值定义下，`DINOv3 + FAISS` 作为跨视角区域级粗定位基线，能够达到怎样的检索表现。

- 数据范围：4 条航线，共 `40` 个 query。
- 卫片候选库：`1029` 张卫片，复用固定候选库与既有 FAISS 索引。
- 方法：`DINOv3` 全局特征 + FAISS `IndexFlatIP`。
- 主展示口径：`top_k=20`；辅助分析口径：`top_k=1029`（全库排序）。

## 2. 新的 Query 定义
本轮实验中的 query 不再沿用旧版 query 集，而是基于 4 条航线重新筛选得到的新一组 `40` 张无人机图像。

- 选取范围：4 条航线，每条航线 `10` 张，共 `40` 张。
- 倾角范围：`-85° ~ -40°`。
- 选取目标：尽量覆盖不同类型地物，同时保持样本在航线内分散，不集中于连续帧。
- 数据使用方式：原始带地理信息图像仅用于 query footprint 恢复和真值生成；实际检索输入使用去元数据后的 query 副本。
- 检索约束：DINOv3 + FAISS 检索阶段不读取 query 的地理坐标、姿态或 EXIF/XMP 信息，只基于图像视觉特征排序。

这一设置更贴近当前任务目标，即验证：在不给检索模型提供 query 地理坐标的前提下，仅依赖单张无人机图像内容，能否把候选区域检索到正确地理范围附近。

## 3. Intersection Truth 定义
本轮正式真值定义为：只要 query 覆盖范围与卫片存在非零面积相交，该卫片就记为 `intersection truth`。
这一定义比原先的单点式真值更宽，也更贴近“只要检索到与查询范围有真实地理交集的区域即可视为有效候选”的任务目标。

## 4. 指标定义
- `Intersection Recall@1/5/10/20`：前 K 名中是否命中 intersection truth。
- `Intersection MRR`：首个 intersection truth 排名倒数的平均值。
- `Top-1 error mean (m)`：首位候选中心与 query 参考位置之间的平均距离。

## 5. 总体定量结果
| 口径 | R@1 | R@5 | R@10 | R@20 | MRR | Top-1误差均值(m) |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |
| Top-20 | 0.775 | 0.950 | 1.000 | 1.000 | 0.850 | 862.191 |
| 全库排序 | 0.775 | 0.950 | 1.000 | 1.000 | 0.850 | 862.191 |

主口径 `Top-20` 下，当前基线达到 `R@1=0.775`、`R@10=1.000`、`R@20=1.000`、`MRR=0.850`。
与全库排序相比，`Top-20` 结果几乎没有损失，说明大部分有效命中已经集中在前 20 名候选中。

## 7. Top-K 曲线结果解读
- full-truth 曲线结果：`40/40` 个 query 都能达到真值饱和，整体 `mean=1028.175`、`median=1029`、`p95=1029`。
- 这说明当前基线在前排候选上的命中能力已经较强，但若希望把全部 intersection truth 全量覆盖到，仍需要接近全库深度。

## 9. 结论
- 在 `intersection truth` 口径下，`DINOv3 + FAISS` 已经具备较强的区域级初步地理定位能力，`Top-20` 下可达到 `R@20=1.000`。
- 当前方法的主要瓶颈不在于前 20 名候选覆盖不足，而在于若希望把所有真值 tile 全部找全，仍需要很深的检索深度。
- 后续若继续优化，应优先提升排序判别力，降低达到全真值覆盖所需的 `K`，而不是仅扩大候选窗口。
