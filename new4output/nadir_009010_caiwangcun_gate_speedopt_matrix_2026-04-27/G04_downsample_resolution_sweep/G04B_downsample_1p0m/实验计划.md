### 问题与结论
| 问题/观察 | 结论 |

| RoMa 阶段不可用 | 1.0m/pix 子组在首航线 RoMa rerank 超过 2 小时未完成，且 stage7 CSV 为 0 字节。 |
| 失败原因 | 记录为 romav2_rerank_timeout_or_cpu_fallback；表现上像 RoMa 没有按预期走可用 GPU 路径或在该输入尺度触发极慢路径。 |
| 没有精度结论 | 由于未进入 PnP 和 validation，不能评价 1.0m/pix 的定位精度，只能记录为工程阻断。 |
| 后续建议 | 若要继续 1.0m/pix，应先做单 pair RoMa GPU smoke test，确认 GPU 进程、输入尺寸、显存和 batch 参数，再决定是否重跑 gate。 |
