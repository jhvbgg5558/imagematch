# 3.1 数据处理流程介绍

## 3.1.1 流程概述

当前卫星 DOM+SRTM 实验的数据处理流程覆盖从 UAV query 准备、固定 satellite DOM 候选库复用、SRTM DSM cache 构建、图像检索、几何重排、SRTM 三维采样、PnP 位姿恢复，到 satellite truth 多层定位精度检验的完整链路。该流程的核心原则是 runtime 阶段不使用 query 地理真值，所有 query truth、AT reference 和 satellite truth patch 只在离线验证阶段使用。

当前正式说明以 `Satellite Truth + SRTM + RoMa v2 Tiepoints` full run 为主。实验根目录为：

`D:\aiproject\imagematch\new3output\nadir_009010_dinov2_romav2_pose_sattruth_srtm_romatie_2026-04-16\`

baseline 与复用资产来源为：

`D:\aiproject\imagematch\new2output\nadir_009010_dinov2_romav2_pose_2026-04-10\`

full run 固定在 `40/40` 个 009/010 nadir query 上。数据处理上复用 baseline 已验证的 selected queries、query inputs、query features、fixed satellite DOM candidate library、DINOv2 retrieval 和 RoMa v2 rerank 资产；重新构建或校验 SRTM DSM cache、formal pose 输入、pose full outputs 和 satellite truth validation outputs。

## 3.1.2 输入数据

### 3.1.2.1 UAV query

query 使用 009/010 两条 UAV 航线中的 nadir 图像，每条航线选择 20 张，共 40 张，query ID 为 `q_001` 至 `q_040`。query 图像在 runtime 检索和位姿求解中被视为不带地理信息的单张 UAV 图像。即使项目中存在 query truth seed 或 AT reference，它们也只用于离线验证、报告和异常分析，不参与候选检索、重排或 PnP 求解。

本路线复用 baseline 中已经锁定的 query-side 资产：

- `selected_queries`
- `query_inputs`
- `query_features`
- `query_truth`

其中 `query_features` 用于 DINOv2 检索，`query_truth` 只用于 offline validation 和 manifest audit。

### 3.1.2.2 固定 satellite DOM candidate library

runtime 候选库为固定 satellite DOM candidate library。该库按 formal 实验口径预先构建，采用 `200m / 300m / 500m / 700m` 多尺度 tile。每个 tile 既是视觉检索阶段的候选影像，也是后续 candidate manifest、DSM request、2D-3D 采样和 PnP 位姿求解的地理载体。

候选库不是在 runtime 中根据单个 query 临时裁剪生成，也不使用 validation truth 来决定候选范围。DINOv2 coarse retrieval 和 RoMa v2 rerank 决定每个 query 的 Top-20 candidate set，query truth 只在离线阶段评估命中情况和定位精度。

### 3.1.2.3 SRTM HGT

DSM 源为锁定的 SRTM HGT 文件：

`D:\aiproject\imagematch\new2output\N30E114.hgt`

该 HGT 是标准 `1 arc-second` SRTM 源数据。formal runtime 不直接消费整块 HGT，而是根据 Top-20 候选中实际出现的 candidate bbox，为每个 active candidate 生成对应的 per-candidate DSM raster。所有 DSM raster 在 formal pose 链路中统一投影为 `EPSG:32650`。

### 3.1.2.4 模型与运行资产

实验使用 DINOv2 进行全局图像特征检索，使用 FAISS 建立候选库索引，使用 RoMa v2 进行 query 与 satellite DOM candidate tile 的局部几何匹配和重排，使用 OpenCV PnP RANSAC 与 refinement 进行粗位姿恢复。

当前 satellite truth + SRTM 路线的主变量不在 runtime 检索和位姿主链，而在 validation 侧：truth orthophoto 改为 satellite truth patch，Layer-3 同名点评估器改为 RoMa v2。runtime 的 DINOv2、RoMa v2 rerank、fixed satellite library、SRTM-backed PnP 主链保持与 baseline 一致。

## 3.1.3 Satellite DOM / SRTM 数据准备

### 3.1.3.1 固定候选库复用

该实验不重新生成 satellite DOM candidate library，而是复用 baseline formal run 中已经建立的 fixed satellite DOM candidate library 和对应 retrieval/rerank 资产。这样做的目的是保持 runtime 主链稳定，只把 satellite truth validation 和 RoMa v2 Layer-3 作为本轮主要验证变量。

复用资产包括：

- fixed satellite library 的 `tiles.csv`
- candidate tile image assets
- query features
- DINOv2 retrieval Top-20
- RoMa v2 reranked Top-20

该约束保证本轮结果可以与 baseline 对比，而不会因为候选库或检索资产变化引入额外变量。

### 3.1.3.2 SRTM HGT 转换与 DSM cache

formal pose 不直接在经纬度 HGT 上做 PnP。处理流程为：

1. 读取 active candidate 的 bbox。
2. 根据 bbox 和扩展 margin 生成 DSM request。
3. 从 `N30E114.hgt` 中裁剪候选区域。
4. 将裁剪结果重投影到 `EPSG:32650`。
5. 写出 per-candidate DSM GeoTIFF。
6. 在 `formal_dsm_manifest.csv` 中绑定 `candidate_id / dsm_id / raster_path`。

关键输出为：

- `pose_v1_formal\dsm_cache\rasters\_summary.json`
- `pose_v1_formal\dsm_cache\rasters\srtm_*.tif`
- `pose_v1_formal\input\formal_dsm_manifest.csv`

当前 full run 的 DSM cache 结果为：

- `planned_count=195`
- `built_count=195`
- `missing_count=0`
- `failed_count=0`
- `status_counts={ready:195}`
- CRS：`EPSG:32650`

这说明所有 active candidate 的 SRTM DSM 请求均成功物化，没有发生 DSM cache 构建失败。

## 3.1.4 Candidate library、特征与检索资产

本路线复用已经完成的 fixed satellite DOM candidate library、candidate DINOv2 features 和 FAISS/retrieval 资产。candidate library 不按 satellite truth patch 或 query truth 临时调整。

DINOv2 粗检索对 40 个 query 输出 Top-20 候选，因此 formal pose 的 query-candidate pair 总数为：

`40 * 20 = 800`

关键路径包括：

- `retrieval\retrieval_top20.csv`
- `romav2_rerank\coarse\retrieval_top20.csv`

这些资产进入 formal pose 后，决定每个 query 需要尝试的候选 tile。后续 PnP 和 scoring 只在这些 Top-20 candidate 内进行，不额外查询 validation truth。

## 3.1.5 RoMa v2 几何重排

DINOv2 粗检索得到的 Top-20 候选进入 RoMa v2 stage7 重排。RoMa v2 在 query 与 candidate satellite DOM tile 之间建立局部几何匹配，输出每个 query-candidate pair 的匹配质量、内点支持和 rerank 排名。

RoMa v2 的作用是把全局特征相似的候选进一步按局部几何一致性排序，使真正地理对应的候选更可能进入 best pose 求解结果。当前 full run 中，RoMa v2 rerank 资产来自 baseline 主链复用，validation truth 的切换不改变 rerank 输入。

## 3.1.6 Formal pose 输入构建

formal pose 阶段需要把 query、candidate、truth audit 字段、DSM cache 和 retrieval Top-20 组织成统一 manifest。关键输出包括：

- `pose_v1_formal\input\formal_query_manifest.csv`

- `pose_v1_formal\input\formal_truth_manifest.csv`
- `pose_v1_formal\input\formal_dsm_manifest.csv`
- `pose_v1_formal\manifest\pose_manifest.json`

其中 `formal_query_manifest.csv` 提供 query 图像路径、近似内参和运行所需元数据；`formal_candidate_manifest.csv` 绑定 satellite DOM candidate tile 图像与地理变换；`formal_dsm_manifest.csv` 绑定每个 candidate 对应的 SRTM DSM raster；`pose_manifest.json` 是 formal pose runtime 的总入口。

需要强调的是，`formal_truth_manifest.csv` 中的 truth 字段只作为 offline audit 和 evaluation 辅助信息，不参与 runtime candidate 选择或 PnP 求解。

### 3.1.8.1 Satellite truth patch 构建

Layer-1 和 Layer-3 使用 satellite truth patch 作为 truth orthophoto。处理流程为：

1. 根据 query truth 和 candidate/truth audit 信息构建 satellite truth manifest。
2. 从 source satellite GeoTIFF 中裁剪 truth patch。
3. 将 truth patch 写入 validation suite 下的 `satellite_truth\truth_patches\`。
4. 记录 `query_satellite_truth_manifest.csv` 和 truth patch summary。

关键输出包括：

- `satellite_truth\query_satellite_truth_manifest.csv`
- `satellite_truth\truth_patches\`
- `satellite_truth\truth_patches\_summary.json`

satellite truth patch 只作为验证对象，不参与 runtime candidate 选择。

### 3.1.8.2 Predicted ortho 生成

predicted ortho 由 best pose、query 图像、candidate-linked SRTM DSM raster 和 truth crop grid 共同生成。它不是多视角 DOM 重建，也不是完整 orthomosaic，而是单张 UAV 图像在 best pose 和 DSM 支撑下的 grid-aligned reprojection product。

关键输出包括：

- `pred_tiles\pred_tile_manifest.csv`
- `pred_tiles\*.tif`
- `pred_tiles\_summary.json`

## 3.1.11 流程复现注意事项

复现实验时最重要的是保持 runtime 主链和 validation 变量边界清晰。fixed satellite DOM candidate library、DINOv2 retrieval、RoMa v2 rerank 和 SRTM DSM cache 是 runtime 主链资产；satellite truth patch、AT reference 和 per-query validation outputs 只能用于 offline evaluation。

不要根据 satellite truth patch 反向修改 candidate library，也不要把 query truth 用于 runtime 候选选择。否则会破坏“单张无地理元数据 UAV 图像定位”的任务定义。

SRTM DSM 需要从 `new2output\N30E114.hgt` 生成 per-candidate DSM cache，并通过 `formal_dsm_manifest.csv` 进入 formal pose。复现时应检查 DSM cache summary，确认 `planned_count == built_count` 且 `failed_count == 0`。

Layer-1 和 Layer-3 属于跨源影像验证。卫星影像和 UAV projected image 在色调、纹理、分辨率和时相上存在差异，因此 `ssim`、tiepoint 数量和 RMSE 都应与 `phase_corr_error_m`、`ortho_iou`、Layer-2 pose-vs-AT 结果联合解释，不能只凭单一指标判断整体定位质量。
