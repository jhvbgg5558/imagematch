# 4.2 定位精度检测方法和指标介绍

## 4.2.1 为什么本节编号为 4.2

当前 DOM+DSM 线路与“卫星 DOM + SRTM”线路都采用三层定位精度验证框架，即：

1. Layer-1：predicted ortho 与 truth orthophoto 的影像级对齐检验；
2. Layer-2：best pose 与 AT/query reference pose 的位姿误差检验；
3. Layer-3：predicted ortho 与 truth orthophoto 的同名点地面误差检验。

因此，两条线路在评价思想、总体层次和许多指标名称上是相似的。但是它们并不是完全相同的定位精度检测方法，主要差异体现在 truth 来源、DSM 来源、Layer-3 同名点匹配器和诊断补充项上。

简言之，两条线框架相似但方法不完全一样；可以共享三层验证的解释结构，但不能把两条线的指标数值视为完全同一口径下的直接对比。

当前 DOM+DSM 线路使用 CaiWangCun 成品 DOM 作为 truth orthophoto，使用 CaiWangCun DSM 作为 PnP 和 predicted ortho 的三维地表支撑，并额外加入 frame sanity 诊断。卫星 DOM + SRTM 线路则使用 satellite truth patch 作为 truth orthophoto，使用 SRTM DSM，Layer-3 使用 RoMa v2 satellite tiepoint evaluator。由于这些差异会影响指标解释和异常归因，本节保存为 `4.2定位精度检测方法和指标介绍.md`，而不是统一写成 `4.定位精度检测方法和指标介绍.md`。

## 4.2.2 两条线路的共同检测框架

两条线路都遵守 runtime truth-free 原则。query 的真实地理位置、truth crop、AT reference pose 只用于离线验证，不参与候选检索、RoMa 重排、DSM 采样或 PnP 求解。定位精度检测发生在 best pose 已经由 runtime 链路求出之后，其目标是回答三个问题：

- predicted ortho 是否落在正确地理区域；
- best pose 的相机中心和视线方向是否接近参考位姿；
- predicted ortho 与 truth orthophoto 的局部细节是否在地面坐标上对齐。

检测输出通常包括每个 query 的 CSV 明细、总体 JSON 汇总、失败桶记录和可视化图件。核心输出文件包括：

- `overall_ortho_accuracy.json`
- `per_query_ortho_accuracy.csv`
- `overall_pose_vs_at.json`
- `per_query_pose_vs_at.csv`
- `overall_tiepoint_ground_error.json`
- `per_query_tiepoint_ground_error.csv`
- `tiepoint_failure_buckets.csv`
- `tiepoints/per_query_matches/<query_id>_tiepoints.csv`

## 4.2.3 DOM+DSM 线路与卫星 DOM+SRTM 线路的差异

下表为两条线路的定位精度检测方法差异表。

| 对比项 | CaiWangCun DOM+DSM full-replacement 线路 | 卫星 DOM+SRTM 线路 |
|---|---|---|
| 当前代表实验 | `nadir_009010_caiwangcun_domdsm_fullreplace_full_2026-04-21` | `nadir_009010_dinov2_romav2_pose_sattruth_srtm_romatie_2026-04-16` |
| runtime 候选库 | CaiWangCun DOM candidate tile library | 固定 satellite DOM candidate library |
| DSM 来源 | CaiWangCun DSM mosaic 裁剪 | SRTM HGT 派生 DSM cache |
| Layer-1 truth | CaiWangCun DOM truth crop | satellite truth patch |
| Layer-1 输出目录 | `eval_pose_validation_suite_caiwangcun_truth/ortho_alignment/` | `eval_pose_validation_suite_sattruth_srtm/ortho_alignment_satellite/`，并镜像兼容到 `ortho_alignment/` |
| Layer-2 reference | AT/query reference pose | AT/query reference pose |
| Layer-3 truth | predicted ortho vs CaiWangCun truth ortho | predicted ortho vs satellite truth patch |
| Layer-3 matcher | orthophoto-truth tiepoint evaluator，当前为 RANSAC inlier 明细 | RoMa v2 satellite tiepoint evaluator |
| 额外诊断 | frame sanity：DSM valid、pred valid、camera/bbox offset、footprint ratio | 主要依赖 satellite truth suite 和报告中的异常样本分析 |
| 指标是否可直接横向比较 | 大部分指标名称相同，但 truth 和 DSM 来源不同，必须带口径解释 | 大部分指标名称相同，但不能直接等同 CaiWangCun DOM truth 口径 |

因此，两条线路可以共享“分层评价”的解释框架，但不能把数值理解为完全同一 truth 条件下的结果。

## 4.2.4 DOM+DSM 线路的正式检测方法

### 4.2.4.1 Layer-1：predicted ortho vs CaiWangCun DOM truth

Layer-1 的目标是检查 best pose 投影出的 predicted ortho 是否与 CaiWangCun DOM truth 在同一地理网格上对齐。处理过程为：

1. 根据 query truth seed 和 CaiWangCun asset manifest 生成 query 对应的 DOM truth crop；
2. 使用 best pose、query 图像、相机内参和 DSM 渲染 predicted ortho；
3. 将 truth ortho 和 predicted ortho 放在同一 truth grid 上；
4. 计算影像级对齐指标和有效覆盖指标；
5. 输出 overlay 可视化图，用于人工检查道路、建筑边界和地物纹理是否系统性扭曲。

主要输出包括：

- `ortho_alignment/query_ortho_truth_manifest.csv`
- `ortho_alignment/truth_tiles/*.tif`
- `ortho_alignment/pred_tiles/*.tif`
- `ortho_alignment/per_query_ortho_accuracy.csv`
- `ortho_alignment/overall_ortho_accuracy.json`
- `ortho_alignment/viz_overlay_truth/`
- `ortho_alignment/viz_overlay_dom/`

### 4.2.4.2 Layer-2：best pose vs AT/query reference

Layer-2 的目标是直接评价 best pose 的相机位置和姿态是否接近参考位姿。该层不评价图像纹理，而是评价 pose 几何本身。

主要输出包括：

- `pose_vs_at/query_reference_pose_manifest.csv`
- `pose_vs_at/per_query_pose_vs_at.csv`
- `pose_vs_at/overall_pose_vs_at.json`

该层与卫星 DOM+SRTM 线路较为接近，因为两者都使用 pose-vs-AT/reference 口径。但是由于两条线路的 runtime DSM、candidate library 和 truth source 不同，Layer-2 结果仍应结合各自的 Layer-1 和 Layer-3 一起解释。

### 4.2.4.3 Layer-3：tiepoint ground error

Layer-3 的目标是评价 predicted ortho 与 truth ortho 的局部细节是否在地面坐标上对齐。DOM+DSM 线路中，Layer-3 在 predicted ortho 与 CaiWangCun DOM truth crop 之间提取同名点，并只保留通过 RANSAC 的有效 inlier。

主要输出包括：

- `tiepoint_ground_error/per_query_tiepoint_ground_error.csv`
- `tiepoint_ground_error/overall_tiepoint_ground_error.json`
- `tiepoint_ground_error/tiepoint_failure_buckets.csv`
- `tiepoint_ground_error/tiepoints/per_query_matches/<query_id>_tiepoints.csv`
- `tiepoint_ground_error/viz_tiepoints/`

每个成功 query 的 `<query_id>_tiepoints.csv` 保存每个同名点的像素坐标、地面坐标和坐标差异。字段包括：

- `query_id`
- `match_index`
- `truth_col_px`
- `truth_row_px`
- `pred_col_px`
- `pred_row_px`
- `truth_x_m`
- `truth_y_m`
- `pred_x_m`
- `pred_y_m`
- `dx_m`
- `dy_m`
- `dxy_m`

坐标差异定义为：

- `dx_m = pred_x_m - truth_x_m`
- `dy_m = pred_y_m - truth_y_m`
- `dxy_m = sqrt(dx_m^2 + dy_m^2)`

该 CSV 的同名点范围与正式 RMSE 指标一致，即只包含 ratio-test 之后被 RANSAC 保留的 inlier，不保存被 RANSAC 剔除的 tentative matches。

### 4.2.4.4 Frame sanity

Frame sanity 是当前 DOM+DSM 线路额外加入的诊断层，用于解释 predicted ortho 异常的来源。它不是新的定位算法，而是基于现有 validation 输出和 DSM cache 进行后验诊断。

主要输出包括：

- `ortho_alignment/frame_sanity/per_query_frame_sanity.csv`
- `ortho_alignment/frame_sanity/overall_frame_sanity.json`
- `ortho_alignment/frame_sanity/failure_buckets.csv`
- `ortho_alignment/frame_sanity/figures/`

Frame sanity 用于回答以下问题：

- DSM 在 truth grid 上是否有效；
- predicted ortho 的有效像素比例是否正常；
- camera center 是否落在 truth bbox 附近；
- predicted valid bbox 与 truth bbox 是否大幅错位；
- truth grid 是否明显大于单视角 query 的真实投影 footprint；
- 异常更可能来自 DSM nodata、pose 错误、坐标框架偏移，还是单视角覆盖不足。

## 4.2.6 指标解释

### 4.2.6.1 Layer-1 影像级对齐指标

`phase_corr_error_m` 表示基于相位相关估计得到的平移对齐误差，单位为米。数值越小，说明 predicted ortho 与 truth ortho 的全局平移偏差越小。该指标主要反映整体平移错位，不直接反映局部扭曲。

`center_offset_m` 表示 predicted ortho 有效区域中心与 truth 参考区域中心之间的地面距离，单位为米。数值越小，说明 predicted ortho 落点越接近 truth 中心。该指标对 500m 级系统性偏移非常敏感，是判断是否存在坐标框架错误的重要指标。

`ortho_iou` 表示 predicted ortho 有效区域与 truth 有效区域的交并比。数值范围通常为 0 到 1，越大表示两者有效覆盖区域重叠越好。该指标既受位姿影响，也受 truth grid 大小和单视角覆盖范围影响。

`ssim` 表示结构相似性，主要衡量 predicted ortho 与 truth ortho 在局部结构、纹理和灰度模式上的相似程度。它对光照、季节、传感器差异和地物变化敏感，因此通常需要和几何指标一起解释。

`valid_pixel_ratio` 或 `pred_valid_pixel_ratio` 表示 predicted ortho 在 truth grid 中的有效像素比例。它不是直接定位误差，但可用于判断 predicted ortho 是否只落在整图很小角落，或者是否因为 DSM/视角覆盖不足产生大面积空洞。

### 4.2.6.2 Layer-2 位姿误差指标

`horizontal_error_m` 表示 best pose 的相机中心水平位置与 reference pose 的水平距离误差，单位为米。它是最直接的位置误差指标之一。

`spatial_error_m` 表示三维空间位置误差，综合水平误差和高程误差。若高程 reference 不稳定，应重点结合 `horizontal_error_m` 解释。

`view_dir_angle_error_deg` 表示 best pose 的视线方向与 reference pose 的夹角误差，单位为度。该指标用于评价相机朝向是否接近真实视线方向。

`yaw_error_deg`、`pitch_error_deg`、`roll_error_deg` 分别表示偏航、俯仰、横滚角误差。它们有助于分析位姿误差主要来自航向偏差、俯仰角偏差还是横滚角偏差。

Layer-2 指标需要结合异常值分布解释。例如当前 DOM+DSM full run 的 Layer-2 mean 被 `q_037` 显著拉高，因此 median 和 p90 更能反映大多数 query 的定位水平。

### 4.2.6.3 Layer-3 同名点地面误差指标

`tiepoint_match_count` 表示同名点匹配数量。数量越多，说明 predicted ortho 与 truth ortho 之间有更多可用于局部几何评价的匹配支撑。

`tiepoint_inlier_count` 表示经过 RANSAC 保留的内点数量。它比原始 match count 更能反映几何一致的有效匹配数量。

`tiepoint_inlier_ratio` 表示内点数与匹配数的比例。数值越高，说明匹配点整体几何一致性越好。

`tiepoint_xy_error_rmse_m` 表示所有有效同名点地面 XY 误差的均方根，单位为米。它是 Layer-3 的核心指标，用于评价局部地物细节在地面坐标中的综合对齐误差。

`tiepoint_xy_error_p90_m` 表示 90 分位同名点地面误差，能反映较大误差尾部情况。相比最大值，p90 对个别极端点不那么敏感。

`dx_m` 和 `dy_m` 分别表示每个同名点在 X/Y 方向上的差异，定义为 predicted minus truth。若大量同名点的 `dx_m`、`dy_m` 方向一致，可能暗示存在系统性平移偏差。`dxy_m` 是该点的水平合成误差。

### 4.2.6.4 Frame sanity 指标

`dsm_sample_valid_ratio_on_truth_grid` 表示 truth grid 上 DSM 可有效采样的比例。若该值很低，则 predicted ortho 的缺失或扭曲可能来自 DSM nodata 或 DSM 覆盖不足。

`dsm_nodata_ratio_on_truth_grid` 表示 DSM nodata 比例，与 DSM valid ratio 互补。该值越高，DSM 支撑风险越大。

`pred_valid_pixel_ratio` 表示 predicted ortho 有效像素占 truth grid 的比例。若该值很低，说明 predicted ortho 只覆盖 truth grid 的很小一部分，需要进一步判断是正常单视角覆盖还是位姿/DSM 异常。

`projected_pixel_count` 表示成功投影到 predicted ortho 的像素数量。该指标可用于判断投影过程是否有足够像素参与。

`camera_center_offset_m` 表示相机中心与 truth 参考区域之间的距离关系。若该值过大，可能说明 best pose 的相机位置不在合理范围。

`bbox_center_delta_m` 表示 predicted valid bbox 与 truth bbox 中心之间的差异。若 bbox 偏移大但 DSM valid 高，通常更倾向于 pose/frame 问题。

`truth_to_footprint_area_ratio` 表示 truth grid 面积与 predicted footprint 面积之间的比例。该值较大时，即使位姿正确，predicted ortho 也可能只覆盖 truth grid 的一部分，因此不能简单把黑边解释为失败。

`pred_valid_bbox_area_ratio` 和 `pred_valid_bbox_aspect_ratio` 用于描述 predicted valid bbox 的面积比例和形状，辅助判断投影是否异常拉伸、变窄或偏角落。

## 4.2.8 指标解读原则

定位精度检测不应只依赖单一指标。`center_offset_m` 可以发现大尺度框架偏移，`horizontal_error_m` 可以评价相机中心位置误差，`tiepoint_xy_error_rmse_m` 可以评价局部细节对齐质量，`pred_valid_pixel_ratio` 和 frame sanity 可以解释 predicted ortho 中黑边或覆盖不足的原因。

当 DSM valid ratio 很高但 `center_offset_m` 或 `horizontal_error_m` 很大时，应优先怀疑 pose/frame 问题，而不是 DSM nodata。当 `pred_valid_pixel_ratio` 较低但 camera center 合理、DSM valid 高、truth-to-footprint area ratio 较大时，应考虑单视角 query 本身无法覆盖完整 truth grid，而不是简单判定定位失败。当多数 query 的偏移方向和幅度接近时，应怀疑 DOM/DSM、candidate library 或 truth/reference frame 混用导致的系统性坐标框架偏移。

当前 CaiWangCun DOM+DSM full-replacement 实验的关键结论是：完整替换后，前期半替换分支中的 500m 级系统性偏移没有复现，predicted ortho 和同名点误差恢复到米级或亚米级可解释范围。因此，当前指标体系不仅用于给出定位精度数值，也用于定位失败原因和验证数据链路是否同源一致。
