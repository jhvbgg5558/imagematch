# 2.1 卫星DOM+SRTM 实验方案介绍

## 2.1.1 实验背景与目标

本实验面向 GNSS 拒止或 GNSS 不可靠条件下的无人机视觉初始定位问题。目标是在 query 图像不携带地理元数据的前提下，仅依赖预先构建的遥感正射卫星影像候选库，为单张 UAV 图像提供区域级检索定位、候选几何重排和粗位姿恢复能力。

当前方法的核心问题可以表述为：给定一张不保证为正射、可能存在尺度变化、光照差异、视角变化和局部遮挡的 UAV query 图像，系统能否仅根据图像内容，在固定 satellite DOM 候选库中找到正确地理区域，并进一步利用 SRTM 提供的地表高程支撑恢复相机粗位姿，最终通过多层验证指标检查定位结果是否稳定。

本轮正式说明以 `Satellite Truth + SRTM + RoMa v2 Tiepoints` full run 为主线。该路线保持 runtime 检索与位姿估计主链不变，即继续使用 DINOv2 粗检索、RoMa v2 重排、固定 satellite DOM 候选库和 SRTM DSM；主要变化发生在 validation 侧：truth orthophoto 改为 satellite truth patch，Layer-3 同名点评估器改为 RoMa v2。该路线已经完成 `40` 张 009/010 nadir query 的 full run，并生成完整报告、三层验证结果、overlay 图件和 per-query 表。

对比 baseline 根目录为：

## 2.1.2 方法总体框架

整体实验链路由“固定 satellite DOM 候选库与 SRTM DSM 准备”和“query 在线式推理/离线评估”两部分组成。

离线阶段复用已构建的固定 satellite DOM candidate library。该候选库采用 `200m / 300m / 500m / 700m` 多尺度 tile 口径，候选 tile 同时承担视觉检索候选和后续位姿求解的地理载体作用。SRTM 数据来自锁定的 HGT 源文件：

`D:\aiproject\imagematch\new2output\N30E114.hgt`

该 HGT 被转换并裁剪为每个 active candidate 对应的 DSM cache raster，统一进入 `EPSG:32650` 坐标系下的 formal pose 链路。

推理阶段对单张 UAV query 图像提取 DINOv2 全局特征，并在 fixed satellite DOM tile 特征库中做 FAISS 检索，得到 Top-20 粗候选。随后使用 RoMa v2 对 query 与每个候选 satellite DOM tile 做局部几何匹配和重排。对于每个 query-candidate pair，系统将候选 DOM 像素映射为投影平面坐标 `(X, Y)`，再从该 candidate 对应的 SRTM DSM raster 中采样高程 `Z`，形成 PnP 所需的 2D-3D 对应关系。最后通过 PnP RANSAC 和 refinement 恢复 query 相机粗位姿，并为每个 query 选择 best pose。

验证阶段不参与 runtime 检索或位姿求解，只用于离线评价。本路线使用 satellite truth patch 作为 Layer-1 和 Layer-3 的 truth orthophoto，对 predicted ortho 与 satellite truth patch 进行图像级对齐和同名点地面误差评估；Layer-2 仍使用 pose-vs-AT/reference 口径，检查估计相机位姿与参考位姿的一致性。

## 2.1.3 核心约束

本方法严格区分 runtime 输入和 offline validation truth。运行阶段不得使用 query 的真实地理位置、query 中心点、AT 参考位姿、satellite truth patch 或 validation crop 来帮助候选选择、候选排序或 PnP 求解。query 图像被视为任意单张 UAV 图像，不要求携带 EXIF/GPS/DJI 地理元数据，也不要求其为正射图像。

卫星 DOM+SRTM 路线保持 runtime 主链不变。也就是说，DINOv2 coarse retrieval、RoMa v2 rerank、fixed satellite DOM candidate library、formal manifests、SRTM DSM cache 和 PnP scoring 仍沿用 009/010 nadir formal pose 口径。本轮实验不把 satellite truth patch 反向用于 runtime 候选库构建，也不在运行时按 query truth 临时裁剪候选。

本实验的主要变量变化集中在 validation 侧：

- truth orthophoto：由 UAV/ODM orthophoto validation 口径切换为 satellite truth patch。
- PnP DSM：保持 SRTM，不切换到 ODM DSM 或 CaiWangCun DSM。
- Layer-3 tiepoint matcher：由早期 SIFT/BF 类口径切换为 RoMa v2 satellite tiepoint evaluator。

因此，本路线适合回答“固定 satellite DOM 候选库 + SRTM DSM 的主链是否能够稳定跑通，并在 satellite truth 口径下获得怎样的定位精度”；不应把它解释为 CaiWangCun DOM/DSM full-replacement 路线，也不应混入 ODM DSM refresh 分支的结论。

## 2.1.4 卫星 DOM 与 SRTM 在方法中的作用

卫星 DOM 主要承担二维视觉参考地图和候选库载体的作用。DINOv2 检索和 RoMa v2 匹配均发生在 query 图像与 satellite DOM tile 之间。候选 tile 的地理仿射变换用于把 satellite DOM 像素转换为投影坐标系下的平面坐标 `(X, Y)`。

SRTM 主要承担低频三维地表支撑作用。RoMa v2 输出的是 query 像素与候选 DOM 像素之间的二维对应关系；若只有 DOM，只能获得平面位置，无法构建 PnP 所需的三维点。引入 SRTM 后，候选 DOM 像素先映射到投影坐标，再从 SRTM DSM 中采样高程 `Z`，从而得到 `(X, Y, Z)` 三维点。

与高分辨率 UAV DSM 或成品城市 DSM 相比，SRTM 的空间分辨率更低，表面更平滑，通常不能刻画屋顶、道路边缘和小尺度地物起伏。但这种低频高程模型也有工程优势：覆盖稳定、nodata 风险较低、局部高程突变更少，能够支撑 formal pose 链路稳定运行。需要注意的是，SRTM 支撑下的 predicted ortho 可能视觉上更平滑，但这不等价于地物几何更精细。

## 2.1.5 主要算法环节

DINOv2 粗检索负责快速从固定 satellite DOM 候选库中召回可能匹配 query 的地理区域。每个 query 输出 Top-20 候选，用于后续几何重排和位姿求解。当前 full run 覆盖 `40` 个 query，因此 Top-20 candidate pair 总数为 `800`。

RoMa v2 几何重排负责在 query 与候选 satellite DOM tile 之间建立局部几何对应，并对 DINOv2 粗检索结果进行重排。RoMa v2 输出的匹配点既用于候选排序，也进入 pose export，用于后续 2D-3D 采样和 PnP。相比只依赖全局特征检索，RoMa v2 能够利用局部几何一致性压低视觉相似但地理错误的候选。

DSM cache 阶段从 SRTM HGT 源文件中为 active candidate 裁剪 DSM raster。当前 full run 的 DSM cache 统计为 `planned=195`，`built=195`，`failed=0`，说明参与 Top-20 pose 求解的所有候选 DSM 请求均成功物化。

DSM 采样阶段把 RoMa 匹配中的 candidate DOM 像素转换为三维地理点。采样状态分为 `ok / out_of_bounds / nodata / unstable_local_height` 等。SRTM 分辨率较低，但在该实验区域覆盖稳定，因此整体上可为 PnP 提供连续高程支撑。

PnP 阶段使用 query 二维点与 SRTM 支撑的三维点执行 `solvePnPRansac`，再进行 refinement。候选位姿会根据内点率、覆盖度、内点数量、高程跨度、重投影误差和位姿合理性惩罚等因素打分，最终为每个 query 选择 best pose。当前 full run 中 full pose 统计为 `best_status_counts={ok:40}`，candidate 级 `score_status_counts={ok:730,pnp_failed:70}`。

## 2.1.6 定位精度验证体系

实验采用多层验证体系，而不是只使用单一误差指标。

Retrieval 指标用于评价候选区域能否被召回，包括 Recall@1、Recall@5、Recall@10、Recall@20 和 MRR。该层回答“正确区域是否进入候选列表”。在本路线中 retrieval/rerank 资产复用 baseline 主链，validation truth 的改变不参与 runtime 检索。

Pose runtime 指标用于评价几何求解是否稳定，包括 DSM cache 成功率、2D-3D sampling 状态、PnP 成功率、best pose score、inlier ratio 和 reprojection error。该层回答“候选区域是否有足够几何支撑用于求解位姿”。

Layer-1 orthophoto alignment 将 predicted ortho 与 satellite truth patch 放在同一验证网格上比较，核心指标包括 `phase_corr_error_m`、`center_offset_m`、`ortho_iou`、`ssim` 和有效覆盖比例。该层回答“由 best pose 投影出来的 query 图像是否与 satellite truth patch 在图像级几何上对齐”。

Layer-2 pose-vs-AT 将 best pose 与 AT/query reference pose 比较，核心指标包括 `horizontal_error_m`、`spatial_error_m`、`view_dir_angle_error_deg`、yaw/pitch/roll error。该层回答“求解出的相机位置和视线方向是否接近参考位姿”。由于 Layer-2 仍使用 pose-vs-AT 语义，因此可以与 baseline 直接对比。

Layer-3 tiepoint ground error 在 predicted ortho 与 satellite truth patch 之间使用 RoMa v2 重新提取同名点，并计算每个同名点在地面坐标中的差异。核心指标包括 `tiepoint_match_count`、`tiepoint_inlier_count`、`tiepoint_inlier_ratio`、`tiepoint_xy_error_rmse_m`、`tiepoint_xy_error_p90_m`。该层回答“局部细节是否在地面坐标上对齐”。

由于本路线的 Layer-1 和 Layer-3 都属于跨源影像对比，`ssim` 和局部纹理相似度会受到卫星影像与 UAV 影像之间的外观域差异影响。因此解释结果时应优先联合 `phase_corr_error_m`、`ortho_iou`、tiepoint RMSE 和 pose-vs-AT 指标，而不是只看单一视觉相似度。

## 2.1.9 结论

当前卫星 DOM+SRTM 路线的核心结论是：在保持 runtime 主链不变的前提下，固定 satellite DOM candidate library 与 SRTM DSM 能够支撑 `40/40` query 的完整 best pose 输出，且 satellite truth validation suite 能够完成 Layer-1、Layer-2、Layer-3 全量评价。

该路线的主要收益集中在验证口径和局部几何评价上。satellite truth patch 使 predicted ortho 与 truth 的有效覆盖重叠和全局平移一致性更稳定；RoMa v2 Layer-3 tiepoint evaluator 显著提升了同名点支持数量，使局部地面误差评估更充分。

同时，实验也表明：更换 validation truth 或提高 Layer-3 matcher 能力，并不等价于直接改善 camera pose 与 AT/reference 的一致性。Layer-2 结果基本保持 baseline 水平或略有回退，说明 runtime 位姿误差仍主要由候选匹配质量、SRTM 高程支撑、PnP 几何条件和参考位姿一致性共同决定。

因此，卫星 DOM+SRTM 路线可以作为“固定遥感正射候选库 + 低频全球 DSM 支撑”的稳定基线。它适合验证在不依赖 UAV DOM/DSM 成品的条件下，系统能否完成单张 UAV 图像的初始地理定位；但如果目标进一步提升城市局部几何精度，仍需要结合更高分辨率 DSM、更一致的 truth 框架和更细粒度的异常 query 分析。
