# 6.2 关键环节耗时统计

本文档统计当前 CaiWangCun DOM+DSM full-replacement full run 的关键运行耗时，并记录运行环境。统计对象为：

`D:\aiproject\imagematch\new3output\nadir_009010_caiwangcun_domdsm_fullreplace_full_2026-04-21\`

需要注意：本次实验不是从零完整重跑所有资产准备环节。full root 中部分 CaiWangCun mosaic、candidate library、candidate features、FAISS、retrieval 和 RoMa rerank 资产来自 gate 分支复用或复制并校验。因此，本文对耗时分两类记录：

- **精确耗时**：日志或 summary 中有明确阶段开始/结束时间，或有内部 `elapsed_seconds`。
- **估算耗时**：只能依据主日志阶段时间和关键输出文件完成时间推算，作为工程耗时参考，不作为严格 benchmark。

## 8. 耗时结论

本轮 full run 的主要耗时集中在两个环节：

1. **Layer-3 tiepoint ground error**：约 `9h15m46s`，是 validation suite 中最大耗时点，也是全链路中最耗时的单个环节。
2. **Full pose pipeline**：约 `5h27m37s`，其中 RoMa pose matches export 和 DSM sampling 是主要耗时来源。

从工程优化角度看，后续优先优化方向是：

- 为 Layer-3 tiepoint evaluation 增加更细的 per-query timing，定位是否由少数 query、图件分辨率、匹配器还是可视化输出主导耗时。
- 对 `4,000,000` 行 DSM sampling 做分块并行、缓存局部 DSM window 或减少中间 CSV I/O。
- 在 orchestrator 中统一记录 `stage_start_utc`、`stage_end_utc`、`elapsed_seconds`、`input_count` 和 `output_count`，避免后续只能依赖文件完成时间估算。
