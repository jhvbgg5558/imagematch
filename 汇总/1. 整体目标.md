# 1. 整体目标

## 1.1 研究背景与问题牵引

无人机在应急救援、灾害监测、国土测绘、城市巡检、低空巡查以及国防安全等任务中，通常需要在复杂环境下持续获得稳定、可靠的位置信息。现有工程系统中，无人机绝对定位主要依赖全球卫星导航系统（GNSS）提供经纬度、高程和速度等导航信息。然而，在城市峡谷、高山峡谷、森林遮挡、复杂电磁环境、地下或半封闭空间以及对抗条件下，GNSS 信号容易受到遮挡、干扰、欺骗或多路径传播影响，导致定位精度下降，甚至出现定位中断和错误定位。对于需要自主执行任务的无人机而言，GNSS 拒止或 GNSS 不可靠会直接削弱航迹保持、任务区域搜索、目标抵近、返航和安全控制能力。

惯性导航系统（INS）和视觉里程计（VO/VIO/SLAM）能够在不依赖外部卫星信号的条件下提供短时间相对运动估计，但它们难以单独解决长期绝对定位问题。INS 误差会随时间积分累积，低成本 IMU 在长航时任务中尤其容易产生漂移；视觉里程计和视觉 SLAM 可以利用连续影像约束相对位姿，但通常缺少稳定的绝对地理坐标约束，单目系统还存在尺度不确定性，在光照变化、纹理贫乏、视角剧烈变化或动态干扰场景中稳定性也会下降。因此，GNSS 拒止环境下的无人机导航不仅需要局部连续运动估计，还需要一种能够周期性提供绝对地理位置约束的视觉全局定位能力。

遥感正射影像和数字地表/高程数据为无人机视觉定位提供了重要的地理先验。正射影像具有明确的地理坐标框架，可作为离线构建的参考地图；DSM/DEM 可提供地面或地表高程约束，为从二维图像匹配进一步恢复三维几何关系提供支撑。但无人机实时影像与遥感正射影像之间存在显著跨视角差异：无人机图像可能是低空、倾斜、局部视角，正射影像则通常为俯视或近俯视成像；二者在尺度、分辨率、光照、季节、纹理表达、遮挡关系和地物外观上均可能不同。这使得传统基于同尺度、同视角、同成像条件的图像匹配方法难以直接满足实际定位需求。

本项目正是在这一背景下展开：在不使用 query 地理元数据、不默认 query 为正射图、不沿用旧同尺度裁块假设的条件下，验证单张无人机图像是否能够依托遥感正射影像候选库完成初始地理定位，并进一步评估能否从区域级检索推进到候选几何重排和粗位姿恢复。该实验是面向 GNSS 拒止环境无人机视觉定位与后续融合导航的基础验证环节，其核心价值在于为 VIO/INS 等连续导航系统提供低频但具备绝对地理意义的视觉观测约束。

## 1.2 本实验的总体目标

本实验的总体目标是：面向 GNSS 拒止或 GNSS 不可靠条件下的无人机自主定位需求，构建并验证一条从单张无人机影像到遥感正射地图候选区域，再到几何重排和粗位姿估计的可复现实验链路，定量评估视觉影像能否支撑无人机初始地理定位及其定位精度检测。

具体而言，本实验首先关注“能否找对区域”。输入 query 被定义为任意单张无人机图像，该图像不携带地理元数据，不保证为正射影像，也不允许依赖旧任务中的 query 中心点真值、同尺度裁块或统一外部分辨率预处理。实验需要在离线预构建的正射影像候选库中，根据图像视觉内容检索出与 query 真实地理覆盖范围相对应的候选区域。该阶段的目标是验证遥感正射影像能否作为视觉参考地图，为单张 UAV 图像提供区域级初始定位能力。

其次，本实验关注“能否把候选排序做得足够可靠”。单纯的全局特征检索可以提供粗候选，但跨视角、尺度差异和地物重复纹理会导致 Top-1 排序不稳定。因此，当前实验链路采用 DINOv2 等视觉特征进行粗检索，再使用 RoMa v2 等局部/稠密几何匹配方法对 Top-K 候选进行重排，目标是在粗召回的基础上提高正确候选进入前排的概率，使候选列表更适合后续几何求解。

再次，本实验关注“能否从区域定位进一步走向粗位姿恢复”。在候选正射影像和 DSM/DEM 支撑下，实验通过图像匹配建立 query 图像二维点与候选地图三维地面/地表点之间的对应关系，再利用 PnP 求解相机粗位姿。该目标并不等同于完整高精度导航系统，也不等同于多视图正射重建，而是验证在检索候选足够接近真实区域时，是否能够获得具有工程意义的初始相机位姿和地理定位结果。

最后，本实验关注“如何检测定位精度”。项目不仅统计检索层面的 Recall@K、MRR 和 Top-1 误差，也进一步建立几何求解和验证层指标，包括匹配内点数量、内点率、重投影误差、PnP 成功率、预测正射图与 truth 图的对齐误差、水平定位误差、同名点地面 XY 误差等。通过多层指标联动，本实验希望把“图像看起来匹配”转化为可量化、可复核、可比较的定位精度检测结果。

## 1.3 无人机定位目标

本实验中的无人机定位目标分为三个层次：区域级候选定位、候选几何精化和粗位姿恢复。

第一层是区域级候选定位。系统需要在离线构建的 DOM/正射候选库中，根据单张 UAV query 图像检索出可能对应的地理区域。该定位结果首先表现为候选 tile 或候选影像块的排序列表，而不是直接输出完整飞行轨迹。由于当前任务强调单张图像、无地理元数据和不保证正射，区域级检索阶段必须保持 truth-free 的运行原则，即检索和排序不能使用 query 的真实位置或真值标签，只能在离线评估时使用真值判断命中情况。

第二层是候选几何精化。DINOv2 粗检索负责快速缩小候选范围，但在跨视角条件下，粗特征相似性并不总能保证最正确的地理区域排在第一位。因此实验进一步引入 RoMa v2 等几何匹配和重排环节，通过候选间的局部对应关系、匹配内点支持、几何一致性和融合分数提升 Top-1 排序可靠性。项目已有结果表明，RoMa v2 重排相对于仅使用粗检索的 baseline 能显著提高前排命中表现，这说明“粗检索召回 + 几何重排纠正排序”是当前视觉定位链路中的关键目标。

第三层是粗位姿恢复。仅获得候选区域还不能直接满足无人机导航系统对位姿观测的需求，因此本实验进一步将候选 DOM 与 DSM/DEM 纳入几何求解。流程上，先在 query 图像与候选正射影像之间建立匹配点，再将候选 DOM 像素点映射到投影坐标系下的平面坐标，并结合 DSM 高程得到三维点，最终通过 `solvePnPRansac` 和后续 refinement 求解无人机相机的粗位姿。该位姿恢复目标强调工程可行性和初值价值，而不是声称已经达到完整实时融合导航系统的最终精度。

当前项目中，009/010 两条 nadir 航线构成了一个锁定的 40-query 实验设置：每条航线选择 20 张近下视 UAV 图像，形成 `q_001` 至 `q_040` 的测试集。该设置用于稳定评估当前链路在真实无人机影像和真实地理影像数据上的表现。实验中，query 图像的地理信息只允许作为离线评估和审计字段使用，不能参与运行时候选选择、候选排序或位姿求解。

## 1.4 定位精度检测目标

本实验的精度检测目标不是单一指标，而是由检索精度、几何匹配质量、位姿求解质量和重投影验证质量组成的多层评价体系。

在检索层，主要检测无人机图像能否找回正确地理区域。核心指标包括 Recall@1、Recall@5、Recall@10、Recall@20、MRR 以及 Top-1 error 等。Recall@K 用于衡量真实候选是否进入前 K 个检索结果，MRR 用于衡量真实候选在排序中的平均靠前程度，Top-1 error 用于描述首位候选与真实位置之间的地理偏差。该层指标回答的是：单张 UAV 图像能否在候选正射库中找到正确或足够接近的区域。

在几何重排层，主要检测候选图像与 query 之间是否存在稳定的视觉对应关系。核心指标包括匹配点数量、RANSAC 内点数量、内点率、匹配覆盖度、候选融合分数和重排前后排名变化。该层指标回答的是：粗检索召回的候选能否被几何一致性进一步筛选和排序，以及错误候选是否能被几何约束压低排名。

在位姿恢复层，主要检测 DOM+DSM+PnP 链路是否能够稳定求解相机粗位姿。核心指标包括 PnP `ok` 数量、`pnp_failed` 数量、有效 2D-3D 点数、DSM 采样成功率、`nodata` 比例、`unstable_local_height` 比例、RANSAC 重投影误差和 refinement 前后误差变化。该层指标回答的是：候选区域不仅是否视觉相似，还是否具备足够的三维几何支撑用于相机位姿初值恢复。

在验证层，主要检测求解出的位姿在地理空间中的实际对齐效果。当前项目使用 predicted ortho 作为重要验证产物：它由每个 query 的 best pose、候选关联 DSM、单张原始 query 图像和 truth crop grid 共同生成，本质是单图像位姿与 DSM 重投影结果，而不是完整的多视图正射重建或 DOM 生产流程。因此，predicted ortho 中出现的局部道路或建筑扭曲，应解释为 DSM 形态、DSM 空洞、插值误差、位姿残差和 truth 裁剪网格共同作用的结果，而不能简单等同为正射重建失败。

验证层的关键指标包括 center offset、horizontal error、phase correlation error、ortho IoU、SSIM、tiepoint XY RMSE、tiepoint p90 误差、frame sanity、valid pixel ratio 和 DSM truth-grid 采样有效率等。它们分别从整体中心偏移、水平定位误差、图像对齐、有效投影覆盖、局部同名点地面误差和坐标框架一致性角度检测定位结果。通过这些指标，本实验能够区分“检索正确但位姿几何不稳定”“DSM 支撑不足”“坐标框架混用导致系统性偏移”“局部视觉差异导致 SSIM 偏低”等不同失败模式。

## 1.5 数据与地图支撑目标

本实验的数据与地图支撑目标，是构建一套运行时 truth-free、评估时可审计、坐标框架一致的候选 DOM/DSM 数据链路。对于 GNSS 拒止环境下的视觉定位而言，地图数据不仅是检索参考，也是后续位姿恢复和精度验证的几何基础。如果候选 DOM、DSM、truth 图、检索特征、重排结果或验证框架来自不同坐标基准且未被统一处理，即使局部匹配质量较好，也可能在最终地理定位层面出现百米级系统偏移。

因此，项目当前强调候选库、候选特征、FAISS 索引、Top-K 检索结果、RoMa v2 重排结果、formal manifests、DSM cache、pose manifest、gate pose 输出和验证 truth 必须保持一致的数据来源和坐标框架。早期 CaiWangCun 诊断分支中，虽然替换了部分 candidate DOM/DSM 并改善了局部匹配质量，但由于仍混用了旧候选或旧检索链路，验证中出现约 515 m 的中心偏移和约 648-652 m 的水平误差。该现象说明，定位实验不能只关注局部视觉匹配，也必须严格控制地图资产、坐标基准和运行链路的一致性。

最新 CaiWangCun DOM/DSM full-replacement gate 分支的目标正是验证这种一致性要求。该分支重建 CaiWangCun DOM/DSM mosaics，并基于 CaiWangCun 数据重新生成 candidate tile library、candidate DINOv2 features、FAISS index、retrieval Top-20、RoMa v2 rerank、formal manifests、DSM cache、pose manifest、gate pose outputs 和 CaiWangCun DOM-truth validation outputs，不再使用 ODM LAZ 或 SRTM fallback。gate 结果显示，候选 tiles 为 149，retrieval `Recall@1 = 0.675`、`Recall@5 = 0.95`、`Recall@10 = 0.975`、`Recall@20 = 0.975`、`MRR = 0.783571`；DSM cache `planned_count = 119`、`built_count = 119`、`failed_count = 0`；PnP 统计为 `{ok: 97, pnp_failed: 3}`，5 个 gate query 的 best pose 均为 `ok`。

更关键的是，该 full-replacement gate 将此前混用链路中的百米级框架偏移恢复到米级验证误差：layer-1 `center_offset_m mean = 4.393887`，layer-2 `horizontal_error_m mean = 1.8294804`，layer-3 `tiepoint_xy_error_rmse_m = 0.3236196`，同时 frame sanity 为 `{ok_or_manual_review: 5}`，DSM truth grid 采样有效率均值约为 `0.9977259`。这说明，当候选 DOM/DSM、检索库、几何重排、位姿求解和验证 truth 在同一框架下重建后，当前实验链路可以从原先的系统性偏移状态恢复到可用于进一步 full run 验证的精度水平。

基于上述结果，数据支撑目标不仅是“拥有高分辨率 DOM 和 DSM”，还包括：候选库边界清晰、DSM 覆盖充分、nodata 和不稳定高程可诊断、query 覆盖状态可追踪、frame sanity 可检查、运行时不泄露 truth、评估时可完整复核。后续 40-query full run 仍需保留这些诊断，以判断 gate 阶段的米级表现是否能够推广到完整样本集。

## 1.6 实验验证与工程意义

从工程意义上看，本实验旨在为 GNSS 拒止环境下的无人机自主导航提供一种可验证的视觉绝对定位观测。它不替代 VIO、INS 或飞控系统，而是为这些连续导航模块提供低频、全局、具有地理坐标意义的外部约束。若单张 UAV 图像能够通过正射地图检索和几何重排稳定找回正确地理区域，并进一步通过 DOM+DSM+PnP 求解出可用粗位姿，则该结果可以作为后续融合导航中的全局观测因子，用于抑制 VIO/INS 长期漂移，修正绝对位置偏差，并增强无人机在 GNSS 不可靠环境中的自主定位能力。

从实验方法上看，本项目将“视觉定位能否可用”拆分为多个可检测问题：候选是否召回、候选是否排在前列、局部几何是否一致、DSM 是否支持 2D-3D 采样、PnP 是否能稳定求解、预测正射投影是否与 truth 对齐、最终水平误差和同名点误差是否处于可接受范围。这样的分层验证方式避免了只凭单一视觉相似性或单一定位误差做结论，也有助于定位失败原因。

从当前项目进度看，实验已经从早期区域级检索逐步推进到更完整的粗位姿恢复验证。DINOv2 粗检索、RoMa v2 重排、SRTM/ODM/CaiWangCun DSM 对比、satellite truth 与 ODM truth 验证、predicted ortho 解释、DSM 分辨率 sweep 和 CaiWangCun full-replacement gate 等分支共同构成了当前方法论基础。其中，ODM DSM sweep 表明 `0.5 m` 是 formal gate 下最高验证支持分辨率，但 `1.0 m` 在视觉稳定性和 nodata 风险上更适合作为实际运行选择；CaiWangCun full-replacement gate 则进一步说明，统一候选 DOM/DSM 与检索链路后，可以显著改善坐标框架一致性和验证误差。

从未来系统闭环看，背景方案中的 VIO、IMU 融合、PX4/Gazebo 仿真和多源融合导航仍属于更完整系统的后续方向。本实验当前不声称已经完成完整融合导航闭环，也不将当前结果等同于实时飞控可用的全系统导航能力。当前阶段的核心贡献在于：建立并验证视觉全局定位与定位精度检测链路，为后续将跨视角视觉定位结果接入 VIO/INS 滑动窗口优化、导航状态估计和仿真飞控闭环提供基础。

## 1.7 阶段性目标与成功判据

本实验的阶段性目标首先是 gate 阶段链路验证。gate 阶段选取具有代表性的少量 query，检查完整流程是否能够无 truth 泄露地完成候选库构建、特征提取、粗检索、RoMa v2 重排、DSM cache 构建、2D-3D 采样、PnP 位姿求解、predicted ortho 渲染和分层验证。gate 阶段的成功判据包括：候选库和 DSM cache 构建无失败，检索 Top-K 具备有效召回，PnP 至少在代表性样本上稳定输出 `ok`，验证 suite `pipeline_status = ok`，且 frame sanity 不出现系统性坐标异常。

第二个阶段目标是 40-query full run 稳定性验证。full run 需要将 gate 阶段已经跑通的链路推广到 009/010 两条 nadir 航线的全部 40 个 query，检查方法是否只在少量样本上有效，还是能够在完整样本集上保持稳定。该阶段应重点统计每个 query 的检索命中情况、RoMa v2 重排效果、PnP 成功状态、DSM 采样状态、predicted ortho 有效像元比例、水平误差、同名点误差和异常样本分布。

第三个阶段目标是异常样本可解释性分析。对于检索失败、Top-1 错误、PnP failed、DSM nodata 过高、unstable local height 过多、predicted ortho 扭曲、SSIM 偏低或 tiepoint RMSE 偏高的样本，实验需要能够判断主要问题来自粗检索召回不足、几何重排失效、DSM 支撑不足、坐标框架不一致、真实跨时相外观差异，还是 query 视角/遮挡导致的视觉不可匹配。只有能够解释失败模式，后续方法改进才具备明确方向。

综合而言，本实验的成功不应只由单个指标决定，而应由以下条件共同支撑：检索层 Recall@K 和 MRR 达到可用水平，正确候选能够稳定进入前排；几何层具有足够匹配内点和合理重投影误差；位姿层 PnP `ok` 比例高且失败原因可诊断；验证层 center offset、horizontal error、tiepoint XY RMSE 等指标进入米级或亚米级趋势；数据层不存在候选 DOM/DSM 与 truth/reference frame 混用造成的系统性偏移；报告层能够清晰区分当前已完成的视觉全局定位/粗位姿验证与未来的 VIO/INS/PX4 融合导航闭环。

在当前进度下，CaiWangCun DOM/DSM full-replacement gate 已经为后续 full run 提供了关键依据：统一数据链路后，系统性偏移可被显著消除，gate 样本上的重投影与地面误差已恢复到可进一步评估的范围。下一步目标应是在保持相同数据一致性和 frame-sanity 诊断的前提下，将 full-replacement 链路扩展到全部 40 个 query，形成更完整的实验统计结论。
