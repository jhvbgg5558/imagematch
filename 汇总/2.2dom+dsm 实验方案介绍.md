# 2.2 DOM+DSM 实验方案介绍

## 2.2.1 实验背景与目标

本实验面向 GNSS 拒止或 GNSS 不可靠条件下的无人机视觉初始定位问题。目标是在不依赖 query 图像地理元数据的前提下，利用已有遥感正射影像（DOM）和数字表面模型（DSM）作为离线地理参考，为单张无人机图像提供区域级检索定位、候选几何重排和粗位姿恢复能力。

当前方法的核心问题可以表述为：给定一张不保证为正射、可能存在视角倾斜、尺度变化、光照差异和局部遮挡的 UAV query 图像，系统能否仅根据图像内容，在预先构建好的 DOM 候选库中找到正确地理区域，并进一步利用 DSM 提供的三维地表高程约束恢复相机粗位姿，最终通过多层验证指标检查定位结果是否达到米级或亚米级可用水平。

本轮正式说明以 CaiWangCun DOM/DSM full-replacement gate 和 full run 为主线。该路线已经完成从 gate 到 40-query full run 的验证，证明当 DOM、DSM、候选库、检索、重排、位姿求解和验证 truth 全部保持同源、同坐标框架时，前期不完整替换实验中出现的 500m 级系统性偏移可以恢复到米级误差范围。

## 2.2.2 方法总体框架

整体实验链路由“候选库离线构建”和“query 在线式推理/离线评估”两部分组成。

离线阶段首先将 CaiWangCun 成品 DOM 和 DSM 切片拼接并统一重投影到 `EPSG:32650`。然后在 DOM/DSM 共同覆盖区域内生成多尺度候选 tile library，tile 尺度采用 `200m / 300m / 500m / 700m`。每个候选 tile 既是检索阶段的视觉候选图像，也是后续 DSM 裁剪、2D-3D 采样和 PnP 位姿恢复的地理载体。

推理阶段对单张 UAV query 图像提取 DINOv2 全局特征，并在候选 DOM tile 特征库中进行 FAISS 检索，获得 Top-20 粗候选。之后使用 RoMa v2 对 query 与每个候选 tile 做局部稠密几何匹配和重排。对于每个 query-candidate pair，系统把候选 DOM 图像上的匹配像素映射到地理平面坐标，并从候选对应的 DSM cache 中采样高程，形成 `(X, Y, Z)` 三维点，再结合 query 图像上的二维匹配点执行 PnP RANSAC 和 refinement，求解 query 相机在地理坐标系中的粗位姿。

验证阶段不参与 runtime 检索或位姿求解，只用于离线评价。系统将 best pose、query 图像和 DSM 投影到 CaiWangCun DOM truth grid 上生成 predicted ortho，并从 orthophoto alignment、pose-vs-AT、tiepoint ground error 和 frame sanity 四个角度检查定位精度和失败原因。

## 2.2.3 核心约束

本方法严格区分 runtime 输入和 offline validation truth。运行阶段不得使用 query 的真实地理位置、query 中心点、AT 参考位姿或 truth crop 来帮助候选选择、候选排序或 PnP 求解。query 图像被视为任意单张 UAV 图像，不要求携带 EXIF/GPS/DJI 地理元数据，也不要求其为正射图像。

当前 CaiWangCun DOM/DSM full-replacement 实验还引入了更强的数据一致性约束：只要替换 DOM/DSM 数据源，所有由候选库派生的资产都必须同步重建，不能只替换 DSM 或只替换 validation truth。必须一起重建的资产包括 candidate tile images、`tiles.csv`、candidate DINOv2 features、FAISS index、DINOv2 retrieval Top-20、RoMa v2 rerank、formal candidate manifest、DSM manifest、DSM cache rasters、pose manifest、pose outputs 和 CaiWangCun DOM-truth validation outputs。

实验明确禁止在 CaiWangCun full-replacement 分支中回退使用 ODM LAZ、ODM DSM、SRTM HGT 或旧 satellite candidate library。这样做的原因是：前期诊断表明，一旦候选库、DSM、truth 或重排结果来自不同坐标框架，局部匹配质量即使看起来较好，最终 predicted ortho 和位姿验证仍可能出现百米级系统偏移。

## 2.2.4 DOM+DSM 在方法中的作用

DOM 主要承担二维视觉参考地图和候选库载体的作用。DINOv2 检索和 RoMa v2 匹配均发生在 query 图像与 DOM tile 之间。DOM 的地理仿射变换用于把候选图像像素转换为投影坐标系下的平面坐标 `(X, Y)`。

DSM 主要承担三维几何支撑作用。RoMa v2 输出的是 query 像素与候选 DOM 像素之间的二维对应关系；若只有 DOM，只能得到平面 `(X, Y)`，无法直接构建 PnP 所需的三维点。引入 DSM 后，候选 DOM 像素点先映射到投影坐标，再在 DSM 中采样高程 `Z`，从而得到 `(X, Y, Z)`。这些三维点与 query 图像上的二维点构成 2D-3D 对应关系，是粗位姿恢复的基础。

因此，DOM 决定“视觉上匹配到哪里”，DSM 决定“这个地理位置的三维支撑是否可用于位姿求解”。如果 DSM nodata 过多、局部高程不稳定或 DSM 与 DOM 坐标框架不一致，PnP 求解和 predicted ortho 都会受到影响。

## 2.2.5 主要算法环节

DINOv2 粗检索负责快速从候选库中召回可能匹配 query 的地理区域。每个 query 输出 Top-20 候选，用于后续几何重排和位姿求解。在当前 full run 中，候选 tile 数为 `149`，retrieval Top-20 输出为 `800` 行，覆盖 `40` 个 query 的全部 Top-20 候选。

RoMa v2 几何重排负责在 query 与候选 DOM tile 之间建立更强的局部几何对应。RoMa 输出的匹配点既用于重新排序候选，也用于后续 2D-3D 采样和 PnP。相比只依赖全局特征检索，RoMa 能够利用局部几何一致性压低视觉相似但地理错误的候选。

DSM 采样阶段把 RoMa 匹配中的候选 DOM 像素转换为三维地理点。采样状态分为 `ok / out_of_bounds / nodata / unstable_local_height`。其中 `unstable_local_height` 表示采样点周围局部高程变化过大或有效邻域不足，不适合进入 PnP。当前 full run 中 2D-3D sampling 统计为 `ok=3,997,663`，`nodata=840`，`unstable_local_height=1,497`，说明 CaiWangCun DSM 在 truth 区域内具有很高的有效支撑。

PnP 阶段使用 query 二维点与 DSM 支撑的三维点执行 `solvePnPRansac`，再进行 refinement。候选位姿会根据内点率、覆盖度、内点数量、高程跨度、重投影误差和位姿合理性惩罚等因素打分，最终为每个 query 选择 best pose。当前 full run 中 PnP candidate 统计为 `ok=781`，`pnp_failed=19`，40 个 query 的 best pose 均为 `ok`。

## 2.2.6 定位精度验证体系

实验采用多层验证体系，而不是只使用单一误差指标。

Retrieval 指标用于评价候选区域能否被召回，包括 Recall@1、Recall@5、Recall@10、Recall@20 和 MRR。该层回答“正确区域是否进入候选列表”。

Pose runtime 指标用于评价几何求解是否稳定，包括 DSM cache 成功率、2D-3D sampling 状态、PnP 成功率、best pose score、inlier ratio 和 reprojection error。该层回答“候选区域是否有足够几何支撑用于求解位姿”。

Layer-1 orthophoto alignment 将 predicted ortho 与 CaiWangCun DOM truth 放在同一 truth grid 上比较，核心指标包括 `center_offset_m`、`ortho_iou`、phase correlation error、SSIM 和 valid pixel ratio。该层回答“由 best pose 投影出来的 query 图像是否落在正确地理位置”。

Layer-2 pose-vs-AT 将 best pose 与 AT/query reference pose 比较，核心指标包括 `horizontal_error_m`、`spatial_error_m`、`view_dir_angle_error_deg`、yaw/pitch/roll error。该层回答“求解出的相机位置和视线方向是否接近参考位姿”。

Layer-3 tiepoint ground error 在 predicted ortho 与 truth ortho 之间提取同名点，计算每个同名点在地面坐标中的差异。成功 query 会输出 `<query_id>_tiepoints.csv`，字段包括 `truth_x_m`、`truth_y_m`、`pred_x_m`、`pred_y_m`、`dx_m`、`dy_m`、`dxy_m`。该层回答“局部细节是否在地面坐标上对齐”。

Frame sanity 用于解释异常，核心指标包括 DSM truth-grid valid ratio、predicted ortho valid pixel ratio、camera center offset、bbox center delta、truth-to-footprint area ratio 等。该层用于区分 DSM nodata、pose 错误、坐标框架偏移、truth grid 过大和单视角覆盖不足等不同失败模式。

## 2.2.9 结论

当前 DOM+DSM 方法的核心结论是：当 CaiWangCun DOM、DSM、candidate library、candidate features、FAISS、retrieval、RoMa rerank、formal manifests、DSM cache、pose manifest 和 validation truth 全部同源重建后，predicted ortho 不再出现前期半替换实验中的系统性倾斜和大面积黑框偏移，整体定位误差从约 500m 级恢复到米级。

这说明前期异常主要不是 CaiWangCun DSM nodata 本身导致，而是旧 candidate/retrieval/rerank 资产与新 DOM/DSM 混用造成的参考框架不一致。当前 full run 结果支持继续沿完整替换路线开展更大范围实验，但 `q_037` 这类异常样本仍需要按 retrieval、RoMa inlier、candidate tile scale、DSM footprint 和 frame sanity 继续分桶诊断。
