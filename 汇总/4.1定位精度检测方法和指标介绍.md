# 4.1 定位精度检测方法和指标介绍

## 4.1.1 为什么本节编号为 4.1

本节对应“卫星 DOM + SRTM”线路的定位精度检测方法和指标介绍。该线路与另一条 DOM+DSM 线路都采用三层验证框架：

1. Layer-1：predicted ortho 与 truth orthophoto 的影像级对齐检验；
2. Layer-2：best pose 与 AT/query reference pose 的位姿误差检验；
3. Layer-3：predicted ortho 与 truth orthophoto 的同名点地面误差检验。

两条线路的评价思想相似，但不是完全相同的检测方法。卫星 DOM+SRTM 线路使用 satellite truth patch 作为验证 truth，使用 SRTM 派生 DSM cache 作为 PnP 和 predicted ortho 的地形支撑，Layer-3 使用 RoMa v2 satellite tiepoint evaluator。由于 truth 来源、DSM 来源和 Layer-3 匹配器口径不同，本节单独保存为 `4.1定位精度检测方法和指标介绍.md`。

## 4.1.2 卫星 DOM+SRTM 线路的检测框架

卫星 DOM+SRTM 线路遵守 runtime truth-free 原则。query 的真实地理位置、AT/reference pose、satellite truth patch 只用于离线验证，不参与候选检索、RoMa v2 重排、DSM 采样、PnP 求解或候选打分。

完整检测发生在 runtime 输出 best pose 之后，核心目标是回答三个问题：

- predicted ortho 是否落在 satellite truth patch 对应的正确地理区域；
- best pose 的相机中心和视线方向是否接近 AT/query reference pose；
- predicted ortho 与 satellite truth patch 的局部地物细节是否在地面坐标中对齐。

## 4.1.3 Layer-1：predicted ortho vs satellite truth patch

Layer-1 的目标是评价 best pose 渲染得到的 predicted ortho 与 satellite truth patch 在同一地理网格上的影像级对齐情况。该层不参与 runtime 位姿求解，只在离线验证阶段使用。

处理流程为：

1. 从 satellite truth suite 构建每个 query 对应的 satellite truth patch；
2. 使用 best pose、query 图像、相机内参和 SRTM DSM 渲染 predicted ortho；
3. 将 predicted ortho 与 satellite truth patch 放到同一验证网格；
4. 计算平移误差、中心偏移、有效区域重叠、纹理相似性等指标；
5. 输出 overlay、outline、pred/truth 对照图，用于人工检查道路、建筑、水系、农田边界等结构是否系统性错位。

## 4.1.4 Layer-2：best pose vs AT/query reference

Layer-2 的目标是直接评价 runtime best pose 的几何精度。该层不比较影像纹理，而是比较 best pose 与 AT/query reference pose 的相机中心、姿态角和视线方向。

处理流程为：

1. 为每个 query 构建 reference pose；
2. 读取 runtime 输出的 per-query best pose；
3. 比较 best pose 与 reference pose 的相机中心坐标；
4. 比较 best pose 与 reference pose 的 yaw、pitch、roll 以及视线方向；
5. 汇总每个 query 的位姿误差和整体统计。

## 4.1.5 Layer-3：RoMa v2 tiepoint ground error

Layer-3 的目标是评价 predicted ortho 与 satellite truth patch 的局部地物细节在地面坐标中的对齐误差。该层使用 RoMa v2 satellite tiepoint evaluator，在 predicted ortho 与 satellite truth patch 之间提取同名点，并基于几何一致的有效匹配计算地面 XY 误差。

处理流程为：

1. 读取 Layer-1 中生成的 predicted ortho 和 satellite truth patch；
2. 使用 RoMa v2 在两幅 ortho 图之间提取同名点；
3. 通过几何约束筛选有效内点；
4. 将同名点像素坐标映射到地面坐标；
5. 计算每个 query 的同名点数量、内点比例、XY 误差统计；
6. 输出 tiepoint overlay 和 error heatmap，辅助判断局部错位的空间分布。

## 4.1.6 指标详细解释

### 4.1.6.1 Layer-1 影像级对齐指标

`phase_corr_error_m` 表示基于相位相关估计得到的全局平移对齐误差，单位为米。数值越小，说明 predicted ortho 与 satellite truth patch 的整体平移偏差越小。该指标主要反映全局平移，不直接反映局部非刚性扭曲。

`center_offset_m` 表示 predicted ortho 有效区域中心与 satellite truth patch 参考中心之间的地面距离，单位为米。该指标对系统性落点偏移敏感，是判断 predicted ortho 是否落在正确区域的重要指标。

`ortho_iou` 表示 predicted ortho 有效区域与 truth 有效区域的交并比。数值越接近 1，说明两者有效覆盖区域重叠越好。它同时受位姿、truth patch 尺寸、单视角覆盖范围和有效像素区域影响。

`ortho_overlap_ratio` 表示 predicted ortho 与 truth patch 的有效重叠比例，用于辅助判断是否存在大面积无效区域或覆盖不足。

`ncc` 表示归一化互相关，用于衡量两幅影像在共同有效区域中的灰度或纹理相关性。跨源影像对比时，该指标会受到光照、季节、传感器响应和重采样影响。

`ssim` 表示结构相似性，主要衡量局部结构、纹理和亮度模式的一致性。卫星 truth 与 UAV predicted ortho 属于跨源对比，外观域差异会显著影响 `ssim`，因此不应把低 `ssim` 单独解释为几何退化证据。

`common_valid_ratio` 表示两幅图在共同验证网格上的共同有效像素比例。该指标用于判断 Layer-1 指标是否有足够的有效区域支撑。

`best_inlier_count` 和 `best_inlier_ratio` 表示影像对齐或局部几何验证中有效内点数量及比例。数量和比例越高，说明对齐判断的几何支撑越充分。

`best_reproj_error` 表示最佳几何模型对应的重投影误差，通常用于辅助判断局部几何一致性。数值越小，说明同名结构在模型约束下越一致。

### 4.1.6.2 Layer-2 位姿误差指标

`dx_m`、`dy_m`、`dz_m` 分别表示 best pose 相机中心相对 reference pose 在 X、Y、Z 方向上的差值，单位为米。通常定义为 best minus reference。

`horizontal_error_m` 表示相机中心水平误差，计算方式为 `sqrt(dx_m^2 + dy_m^2)`。它是最直接的位置精度指标之一，适合用于描述平面定位精度。

`spatial_error_m` 表示相机中心三维空间误差，计算方式为 `sqrt(dx_m^2 + dy_m^2 + dz_m^2)`。当高程 reference 或 DSM 高程存在不确定性时，应结合 `horizontal_error_m` 一起解释。

`view_dir_angle_error_deg` 表示 best pose 的视线方向与 reference pose 的视线方向夹角，单位为度。该指标用于评价相机朝向是否接近参考视线方向。

`yaw_error_deg`、`pitch_error_deg`、`roll_error_deg` 分别表示偏航、俯仰、横滚角误差。它们用于诊断姿态误差主要来自航向偏差、俯仰偏差还是横滚偏差。

`best_score` 是 runtime 候选打分结果，用于解释为什么某个候选被选为 best pose。它不是直接的定位误差，但可用于分析低精度 query 是否对应低置信度候选。

### 4.1.6.3 Layer-3 同名点地面误差指标

`tiepoint_match_count` 表示 predicted ortho 与 satellite truth patch 之间提取到的同名点匹配数量。数量越多，说明局部评价有更多匹配支撑。

`tiepoint_inlier_count` 表示通过几何一致性筛选后保留的内点数量。它比原始 match count 更能反映可用于误差统计的有效匹配点数量。

`tiepoint_inlier_ratio` 表示内点数与匹配数的比例。数值越高，说明同名点整体几何一致性越强。

`tiepoint_xy_error_mean_m` 表示有效同名点地面 XY 误差的平均值，单位为米。

`tiepoint_xy_error_median_m` 表示有效同名点地面 XY 误差的中位数。相比 mean，它对少量大误差点不那么敏感。

`tiepoint_xy_error_rmse_m` 表示有效同名点地面 XY 误差的均方根，单位为米。该指标对大误差点更敏感，是 Layer-3 的核心综合指标。

`tiepoint_xy_error_p90_m` 表示 90 分位同名点地面误差，用于描述误差尾部。相比最大值，它更适合反映多数匹配点之外的较大误差水平。

`tiepoint_xy_error_max_m` 表示最大同名点地面误差。该指标可用于定位极端异常点，但不宜单独代表整体精度。

`tiepoint_dx_mean_m` 和 `tiepoint_dy_mean_m` 表示同名点在 X/Y 方向的平均偏差。若它们在多个 query 中方向一致，可能说明存在系统性平移偏差；若方向分散，则更可能是局部匹配、地物变化或视角差异造成的误差。

## 4.1.8 指标解读原则

第一，三层指标需要联合解释。Layer-1 说明 predicted ortho 与 satellite truth patch 的影像对齐情况，Layer-2 说明 runtime best pose 与 reference pose 的几何差距，Layer-3 说明局部同名点在地面坐标中的细节误差。任何单层指标都不应被单独用来给出完整定位结论。

第二，`ssim` 在卫星 truth 与 UAV predicted ortho 的跨源对比中受外观域差异影响明显。光照、季节、传感器响应、影像锐度和重采样方式都会影响 `ssim`，因此它更适合作为辅助指标，而不是单独的几何精度判据。

第三，Layer-3 的 `tiepoint_match_count` 和 `tiepoint_inlier_count` 提升，说明局部几何评价有更充分的同名点支撑，但不保证每个 query 的 `tiepoint_xy_error_rmse_m` 都同步下降。匹配点数量、地物重复纹理、局部遮挡、视角差异和 truth patch 外观变化都会影响最终 RMSE。

第四，Layer-2 的 pose-vs-AT/reference 是对 runtime best pose 本身的评价。它主要反映检索候选、RoMa 重排、SRTM DSM 采样、PnP 求解和候选打分的综合结果，不会因为 validation truth 改成 satellite patch 自动改善。

第五，卫星 DOM+SRTM 线路的数值应带口径解释。其 truth patch、DSM 来源和 Layer-3 matcher 与其他线路不同，因此可以比较趋势和诊断结论，但不应把所有数值当作完全同一 truth 条件下的直接横向对比。
